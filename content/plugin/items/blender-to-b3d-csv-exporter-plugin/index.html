This document provides a line-specific breakdown of all changes made to the Blender OpenBVE Addon, identifying what was added, modified, or deleted, along with the technical and design rationale.

You can find the original version of this plugin at: Original Version: https://github.com/maisvendoo/blenderCSV Post 2.80+ Version: https://github.com/zbx1425/blenderCSV b3d exporter version: https://progressiveaddon.weebly.com

=========================================================================== FILE: __init__.py

[MODIFIED] Line 22-29 (Original Version):

  • Change: Version (0, 7, 0) -> (0, 8, 0).
  • Rationale: Surpasses the original release to signify significant feature   additions (B3D, Author tagging) and a major architectural shift (Exporter-only).

[MODIFIED] Line 32 (Original Version):

  • Change: "blender": (4, 0, 0) -> (2, 80, 0).
  • Rationale: Increased compatibility. While the code is optimized for 4.x+,   this allows users on older stable LTS versions (2.83, 3.6) to still use it.

[DELETED] Line 51-95 (Original Version):

  • Change: Removed the entire `CsvImporter` class.
  • Rationale: Strategic decision to deprecate the importer. Removing this large   code block (along with `ImportCSV.py`) protects the game community's assets   from unauthorized conversion while reducing addon bloat.

[MODIFIED] Line 98-121 (Original Version / Exporter Operator):

  • ADDED (L107-121): `update_export_format` and `export_format` EnumProperty.
  • ADDED (L123-132): `texture_path_mode` (Absolute, Relative, Filename Only).
  • ADDED (L134-138): `author_name` Metadata field.
  • Rationale: New features requested to handle .b3d format, customizable path   handling for different asset structures, and creator attribution.

[DELETED] Lines 285-286, 294-295, 312-313 (Original Version):

  • Change: Removed `menu_import` append and `CsvImporter` registration.
  • Rationale: Complete UI cleanup after removing the importer backend.

=========================================================================== FILE: CSV.py

[DELETED] Line 45-50 (Original Version):

  • Change: Removed `ImportOption` and standard `logging` import.
  • Rationale: Part of the importer removal; shifted to using the internal   `.logger` module for consistency.

[MODIFIED] Line 51-57 (Original Version / ExportOption):

  • Change: Added `export_format`, `texture_path_mode`, and `author_name`.
  • Rationale: Centralized configuration state for the new exporter features.

[DELETED] Lines 199-926 (Original Version):

  • Change: Deleted the ~727 line `load_csv` method.
  • Rationale: This was the core parsing engine for imports. Its removal is the   primary part of the "clean-out" to make this an Exporter-only plugin.

[MODIFIED] Lines 928-1007 (Original Version / export_csv):

  • Change: Complete refactor from simple string concatenation to a list-based   buffer system (`lines = []`).
  • ADDED: `f4()` helper (L268-272) for 4-decimal precision and stripping Zeros.
  • ADDED: `fmt_cmd()` helper (L263-266) to handle B3D vs CSV syntax differences.
  • Rationale: PERFORMANCE and PRECISION. Joins are faster than repeated string   addition. The `f4` helper prevents scientific notation (like 1e-05) which   crashes some legacy BVE parsers.

=========================================================================== FILE: ExportCSV.py

[ADDED] Current Lines 69-100:

  • Change: New helper `_get_processed_texture_path`.
  • Rationale: Centralizes path logic. It handles the "Filename Only" request   by using `.name`, and ensures all paths use forward slashes (`/`) for   OpenBVE cross-platform compatibility.

[MODIFIED] Line 93-112 (Original Version / Vertex Logic):

  • Change: Implemented `vertex_cache` (L139-169 in current version).
  • Rationale: MAJOR OPTIMIZATION. The original code added a new vertex to the   list for every face corner. The new cache merges vertices with identical   pos/normal/uv data, reducing file size and vertex count by 60-80% for   typical models.

[MODIFIED] Line 132-152 (Original Version / Material Logic):

  • Change: Replaced hardcoded node index with a robust generator search:   `next((n for n in mat.node_tree.nodes if n.type == 'BSDF_PRINCIPLED'), None)`.
  • Rationale: CRASH PREVENTION. The original would crash if a material didn't   have exactly one Principled BSDF node in a specific order. The new logic is   fail-safe and handles static colors on the input even without texture nodes.

=========================================================================== FILE: Transform.py

[MODIFIED] Line 38 and 46 (Original Version):

  • Change: Normal transformation logic.
  • Original: `(swap_mat @ matrix_world).to_3x3().normalized()`
  • Current: `full_transform.to_3x3().inverted().transposed()`
  • Rationale: MATHEMATICAL CORRECTNESS. Using the Inverse-Transpose matrix is   the standard for transforming normals when non-uniform scaling (e.g., stretching   on X only) is applied, ensuring lighting remains correct.

=========================================================================== FILE: ImportCSV.py

[DELETED] (Total File Removal)

  • Rationale: This file was dedicated entirely to the Importer. Removing it   ensures no residual code path can be used to re-enable import capabilities.
